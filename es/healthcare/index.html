FHIR: Recursos de Interoperabilidad Rápida de Atención Médica.

El estándar de Recursos de Interoperabilidad Rápida de Atención Médica (FHIR) es un estándar de atención médica digital diseñado para facilitar el intercambio de registros médicos electrónicos (RME) entre diferentes sistemas de atención médica. Es un componente clave del ecosistema de atención médica moderna, que permite a los proveedores de atención médica compartir datos de pacientes de manera efectiva y colaborar de manera más eficiente.

FHIR se basa en los últimos estándares web y está diseñado para ser flexible, escalable y fácil de implementar. Incluye una variedad de API, modelos de datos y protocolos de seguridad que permiten que los sistemas de atención médica se comuniquen e intercambien datos de manera segura y estandarizada. Esto ayuda a mejorar la eficiencia de la prestación de atención médica, reducir el riesgo de errores y aumentar la seguridad del paciente.
Además de sus capacidades técnicas, FHIR también cuenta con una sólida comunidad de desarrolladores y profesionales de la salud que participan activamente en la configuración del estándar. Esto garantiza que siga siendo relevante y receptivo a las necesidades de la industria de la salud.

Arquitectura del Sistema y de Datos.

Aplicación de Toma de Decisiones Compartidas (ATDC): Una Aplicación de Internet Rica (RIA) basada en la web que respalda a los profesionales de la salud (médicos y enfermeras de apoyo) en la fase de toma de decisiones compartidas.
Aplicación Móvil SMSS
Aplicación Web SMSS: Una aplicación web que también respalda al paciente durante los períodos de autogestión. Complementa la Aplicación Móvil y permite al paciente gestionar sus planes de acción de autogestión semanalmente o quincenalmente a través de pantallas más grandes.
Interfaz Web de Administración de Usuarios y Privacidad: Una aplicación web que maneja las operaciones administrativas comunes de administración de usuarios y privacidad.
En la capa de servicios de backend, se encuentran los siguientes componentes principales:
Almacén de Datos Personales: El componente que proporciona un almacenamiento seguro para todos los datos de salud del paciente que deben registrarse en el sistema y proporciona un servicio RESTful seguro basado en HL7 FHIR para que otros componentes accedan y actualicen los datos registrados.
Servicios Principales: El componente que proporciona los servicios principales, como autenticación, identidad de usuario y administración de autorizaciones, y almacenamiento de registros de auditoría.
Servicios de Predicción: Todos los modelos predictivos a corto, mediano y largo plazo utilizados en KADIS, MT2D-MARVEL y los modelos de puntuación de riesgo se exponen como servicios de una manera que es consumible por otros componentes relevantes a través de una API web o a través de las API correspondientes dentro de la plataforma de backend.
Motor de Planes de Acción: El servicio de inteligencia de backend para las aplicaciones móviles y web SMSS.
